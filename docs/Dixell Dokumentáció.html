PROGRAM DOKUMENTÁCIÓ

 

TELEPÍTÉSI ÚTMUTATÓ

 

1.    Szükséges csomagok telepítése

 

Linux:

Nodejs: sudo apt install nodejs

npm: sudo apt install npm

 

2.    Program letöltése és kicsomagolása

 

Letöltés:

A program 2 mappát tartalmaz: “riasztasok” és “homerseklet”. Ez tartalmazza az összes szükséges komponenst. Ezeket kell kicsomagolni a használni kívánt mappába.

 

Telepítés:

Terminal-al bele kell menni a “riasztasok” mappába és kiadni az npm install parancsot. Ez telepíti a szükséges nodejs csomagokat.

 

3.    Program futtatása

 

A program futtatása a node backend.js paranccsal történik. Ahhoz, hogy ez automatikusan történjen a számítógép indításakor, ahhoz az alábbi utasításokat kell kiadni:

 

sudo nano /etc/systemd/system/myscript.service

myscript.service tartalma:

 

[Unit]

Description=Dixel Service

 

[Service]

User=pi

ExecStart=/usr/bin/node /path/to/file/riasztasok/backend.js

 

[Install]

WantedBy=multi-user.target

 

ahol a /path/to/file a “riasztasok” mappa útvonalát jelenti.

 

Ezután a service indítása:

sudo systemctl enable myscript

sudo systemctl start myscript

 

Megnézni, hogy a program fut-e a következő paranccsal lehet:

 

sudo systemctl status myscript

 

4.    Weboldal megnyitása

 

Ha a backend.js indítása megtörtént, el lehet indítani a weboldalakat. A két weboldal: “riasztasok/index.html” és “homerseklet/index.html”.

 

 

Használati útmutató

 

A weboldalakat csak akkor kell megnyitni, ha már elindult a háttérben a node backend.js

program. Ha a Telepítési útmutatóban leírt automatikus indítás meg lett csinálva, akkor ezzel nem . A weboldalak maguktól működnek, 4 gomb van, amivel lehet szabályozni őket:

 

 

Ezek mind a “riasztasok” mappában lévő oldalon vannak. Az első gomb indítja el a Dixel oldalak lekérdezését. A második gombbal lehet megállítani és a harmadik gombbal lehet újraindítani. A negyedik gombbal lehet a sípolásokat némítani, illetve felhangosítani.


 

Hiba észlelése

 

Mind a 2 weboldal írja, hogy mikor volt legutóbb frissítve. Ha egy ideje nem frissít, akkor valószínűleg valami hiba van a rendszerben.

 

Logok a journalctl -u myscript.service paranccsal érhetők el

 

Fájl Struktúra

 

1.    Hőmérséklet

 

Hőmérséklet mappa

 

DIXELL HŐFOKÖSSZESÍTŐ.HTML:

 

HTML fájl, azon kívül, hogy be includolja a Javascript és css fájlokat, nem csinál semmit.

 

SCRIPT1.JS:

 

Dropdown lekezelő JS fájl.

 

SCRIPT2.JS:

 

Itt hozzuk létre a weboldal kinézetét. Illetve kezeljük le a WebSocket-en lévő üzeneteket, azaz a frissítéseket is.

 

STYLE:CSS:

 

Kinézetet leíró fájl.

 

 

2.    Riasztások

 

Riasztások mappa

 

 

INDEX.HTML:

 

A Dixell Weboldalakhoz hasonló struktúra teljesen lemásolva. Illetve hozzáadva pár gombot és audio-t. A gombok egy http-requestet küldenek a Backend-nek. Van még hozzá írva egy kis JS rész, ahol a WS üzeneteket kezeljük le, illetve még egy rész, ahol a gomboknak az event-jeit dolgozzuk fel.

 

BACKEND.JS:

 

A server. Fő része a programnak. Ezt mindig el kell indítani, mielőtt akarunk valamit megjeleníteni. Van generálva egy JSDoc fájl, ami dokumentálja.

 

STYLE.CSS:

 

Kinézetet leíró fájl.

 

BEEP.MP3:

 

Sípoló hang audio fájl-ja

 

INPUT.JSON:

 

Sorrendet megadó JSON fájl. Könnyen érthető/olvasható.

 

JSDOC.JSON:

 

Doksi konfiguráció

 

PACKAGE.JSON:

 

Boilerplate. Generált fájl. Hozzáírtam még 1 scriptet, ami a doksit generálja. (doc => futtatása: npm run doc)

 

PACKAGE-LOCK.JSON:

 

Boilerplate. Generált fájl.
